# BV2URL

基于 Cloudflare Workers 和 Hono 框架构建的 Bilibili (B站) 视频解析服务。主要功能是将 Bilibili 的 BV 号转换为可以直接播放的视频直链。

## ✨ 核心业务逻辑与架构

由于 Cloudflare Workers 的 IP 段普遍被 B站风控拦截，本项目采用了**边缘节点 + 独立反代**的架构：

1. **URL 解析与路由分发**：由部署在 Cloudflare Workers 上的 Hono 框架负责，提供极低的访问延迟和高并发处理能力。
2. **反代请求中转**：实际的 B站 API 请求会通过 `ky` 客户端转发至部署在 Render 上的专属代理服务，从而成功绕过 IP 限制获取真实视频流地址。
3. **保活机制 (Keep-Alive)**：由于 Render 的免费实例会在无请求时休眠，本项目在 Worker 中配置了定时触发器（Cron Trigger），定时向 Render 发起网络请求以保持反代服务处于活跃状态。

## ⚙️ 环境配置说明

本项目的环境变量严格分为**公开配置**与**私密鉴权**两部分。

### 1. 公开环境变量 (正常变量)

这些变量定义在 `wrangler.jsonc` 的 `[vars]` 块中，随代码库一起进行版本控制。

| 变量名        | 说明                               |
| :------------ | :--------------------------------- |
| `CORS_ORIGIN` | 允许跨域请求的前端白名单地址       |
| `BILI_PROXY`  | 部署在 Render 上的反代服务完整 URL |

### 2. 私密环境变量 (Secrets)

包含账户凭证与鉴权 Token，**绝对不可**硬编码在代码或 `wrangler.jsonc` 中。
项目本地开发时，在根目录创建一个名为 \`.dev.vars\` 的文件，并在其中填入你的私密变量。

| 变量名        | 说明                                                         |
| :------------ | :----------------------------------------------------------- |
| `BILI_COOKIE` | 用于获取高清晰度视频流的 B站账号 Cookie                      |
| `PROXY_TOKEN` | 与 Render 反代服务通信的自定义鉴权 Token，防止代理被恶意盗用 |
